# Решение проблем с SSL в Telegram Bot

## Проблема

При запуске бота может возникать ошибка:
```
ClientConnectorSSLError: Cannot connect to host api.telegram.org:443 ssl:default 
[SSL: WRONG_VERSION_NUMBER] wrong version number
```

## Причины

Эта ошибка обычно возникает при:
1. Работе через прокси-сервер, который перехватывает HTTPS соединения
2. Наличии корпоративного файрвола с SSL inspection
3. Проблемах с SSL сертификатами в системе

## Решения

### Вариант 1: Отключить проверку SSL (только для разработки!)

⚠️ **ВНИМАНИЕ**: Это небезопасно и должно использоваться только для локальной разработки!

Добавьте в файл `.env`:
```bash
TG_DISABLE_SSL_VERIFY=1
```

### Вариант 2: Использовать кастомный CA файл

Если у вас есть корпоративный сертификат, укажите путь к нему:

```bash
TG_SSL_CA_FILE=/path/to/your/ca_bundle.pem
```

### Вариант 3: Установить certifi

Убедитесь, что установлен пакет `certifi`:
```bash
pip install certifi
```

Он уже добавлен в `requirements.txt`.

## Технические детали

Бот использует кастомную aiohttp сессию (`botapp/tg_session.py`), которая:
- Автоматически настраивает SSL контекст на основе переменных окружения
- Использует `certifi` для получения актуальных CA сертификатов
- Позволяет отключить проверку SSL для разработки
- Поддерживает кастомные CA файлы

## Проверка

После настройки перезапустите бота:
```bash
python run_local.py
```

Если ошибка сохраняется, проверьте:
1. Правильность значения переменной окружения в `.env`
2. Доступность `api.telegram.org` через прокси (если используется)
3. Логи бота на наличие предупреждений об SSL
