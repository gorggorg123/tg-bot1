# 📚 Примеры использования Ozon API

Эта директория содержит практические примеры использования модернизированного Ozon API клиента.

## 📁 Файлы

### `ozonapi_usage.py`

Комплексный файл с примерами всех основных возможностей нового API клиента.

**Что демонстрируется:**

1. ✅ Получение каталога товаров (с автоматической пагинацией)
2. ✅ Batch получение информации о товарах (с кешированием)
3. ✅ Получение FBO заказов за период
4. ✅ Работа со складами
5. ✅ Работа с чатами (V3 API)
6. ✅ Получение финансовых данных
7. ✅ Получение аналитики (с кешированием)
8. ✅ Просмотр метрик производительности

## 🚀 Как запустить

### Шаг 1: Настройка переменных окружения

Создайте файл `.env` в корне проекта (если еще не создан) и добавьте:

```bash
OZON_CLIENT_ID=your_client_id
OZON_API_KEY=your_api_key

# Опционально (рекомендуется)
OZON_MAX_RPS=20
OZON_ENABLE_CACHE=true
LOG_LEVEL=INFO
```

### Шаг 2: Установка зависимостей

```bash
pip install -r requirements.txt
```

### Шаг 3: Запуск примеров

```bash
# Из корня проекта
python examples/ozonapi_usage.py
```

## 🔧 Настройка примеров

### Изменение периода для заказов/финансов

Отредактируйте `ozonapi_usage.py`:

```python
# Вместо последних 7 дней
date_from = date_to - timedelta(days=7)

# Можно использовать любой период
date_from = date_to - timedelta(days=30)  # Последний месяц
date_from = date_to - timedelta(days=1)   # Последний день
```

### Изменение количества товаров

```python
# В example_get_catalog()
products = await client.get_product_list_all(
    visibility='VISIBLE',
    limit_per_page=1000,  # Изменить лимит на страницу
)
```

### Включение/выключение кеша

```python
# Включить кеширование (рекомендуется)
use_cache=True

# Отключить (для тестирования)
use_cache=False
```

## 📝 Создание собственных примеров

Используйте существующие примеры как шаблон:

```python
from botapp.ozon_api_client import get_ozon_client
from botapp.logging_config import setup_logging, log_performance

setup_logging(log_level="INFO")

@log_performance("мой_пример")
async def my_example():
    """Мой пример использования API"""
    client = get_ozon_client()
    
    # Ваш код здесь
    products = await client.get_product_list_all()
    
    logger.info("Получено %s товаров", len(products))
    
    return products

# Запуск
import asyncio
asyncio.run(my_example())
```

## ⚠️ Важные замечания

### 1. Rate Limiting

Примеры используют настройки rate limiting из `.env`. При большом количестве запросов может потребоваться время.

**Если получаете ошибку 429** (Rate Limit Exceeded):

```bash
# В .env уменьшите RPS
OZON_MAX_RPS=15  # Было 20
```

### 2. Требуемые права

Некоторые примеры требуют специальных прав:

- **Аналитика** - требует роли с доступом к аналитике
- **Чаты** - может требовать Premium+ подписку
- **Отзывы/Вопросы** - требуют прав на работу с отзывами/вопросами

Если получаете ошибку прав доступа, это нормально - пример просто пропустится.

### 3. Пустые результаты

Если у вас новый аккаунт или мало данных, некоторые примеры могут возвращать пустые списки:

```
[INFO] Получено товаров: 0
[WARNING] Нет товаров для примера
```

Это нормально - просто добавьте товары/заказы в ваш аккаунт Ozon.

## 🐛 Troubleshooting

### Проблема: "OZON_CLIENT_ID не установлен"

**Решение**: Создайте файл `.env` с переменными:

```bash
OZON_CLIENT_ID=your_client_id
OZON_API_KEY=your_api_key
```

### Проблема: "ModuleNotFoundError"

**Решение**: Установите зависимости:

```bash
pip install -r requirements.txt
```

### Проблема: "Получено 0 товаров" для всех примеров

**Решение**: 
1. Проверьте правильность `OZON_CLIENT_ID` и `OZON_API_KEY`
2. Убедитесь что в аккаунте есть товары/заказы
3. Проверьте права API ключа в личном кабинете Ozon

### Проблема: Медленное выполнение

**Решение**:
1. Включите кеширование: `OZON_ENABLE_CACHE=true`
2. Уменьшите количество запросов в примерах
3. Проверьте скорость интернет-соединения

### Проблема: Ошибки SSL

**Решение**: См. [SSL_TROUBLESHOOTING.md](../docs/SSL_TROUBLESHOOTING.md)
